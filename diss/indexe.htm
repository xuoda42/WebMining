Introduction

This work focuses on developing a web application for managing projects and tasks in small and medium-sized teams. The relevance of the topic is driven by the growing complexity of projects, the spread of remote/hybrid work, and the fact that a significant share of working time goes to coordination and 'work about work' rather than to completing tasks. Against this background, tools that provide process transparency and convenient real-time task management become critically important.

1 Goal and main idea of the project

The goal is to create an intelligent task planner with a modern interface and task-management features comparable to popular systems, but more focused on ease of adoption and practical value. The key 'intelligence' is implemented as a logic module that controls task transitions between states (essentially, a finite-state machine with allowed transitions). This reduces the number of logical errors in the workflow and defines formalized rules for moving tasks through stages.

2 Application functionality

As a result, a web application was developed that supports user registration and login (authentication), creating, editing, and tracking tasks, assigning executors and setting priorities, working with a Kanban board with customizable columns and moving tasks between columns via drag-and-drop, as well as a responsive interface for different devices.

3 Selected technologies and architectural decisions

The server side is built with Python + FastAPI, which provides a fast REST API and a convenient development structure. For data access, an ORM approach is used via SQLAlchemy, and PostgreSQL was chosen as the DBMS (reliability, transactional consistency, and scalability for multi-user work). The client side is implemented with plain HTML/CSS/JavaScript without heavy frameworks—this lowers the deployment threshold and reduces the application's 'weight' while preserving interactivity (fetch requests, DOM manipulation, events, async/await).

4 Development, integration, and performance evaluation

The development process is described—from designing the data models (user, task, column) to implementing the API and the interface. For security, JWT tokens and password hashing (bcrypt) are used. An analytics block is also considered separately: productivity metrics are proposed (for example, time to create the first project, error frequency, and coordination overhead) along with improvements through hints, validation, and assistance mechanisms (Smart Help). Emphasis is placed on reducing onboarding time and speeding up the resolution of typical user issues.

5 Conclusions

The work results in a full-featured intelligent task planner implemented as a web application: with a Kanban board, drag-and-drop, authentication, and a server architecture based on FastAPI + PostgreSQL. A key feature is the module of formalized 'intelligent' logic that controls the correctness of task transitions between states, thereby increasing the reliability of the task management process. Further development potential is also outlined: expanded project management (deadlines, reminders, subtasks), analytical dashboards and forecasting, integrations with external services, and further scalability optimization.
